คู่มือการใช้งานระบบจัดการร้านซักอบรีดอัจฉริยะ

📘 คู่มือการใช้งานระบบ (User Manual)

ระบบ Washly ถูกออกแบบมาเพื่ออำนวยความสะดวกในการจัดการร้านซักอบรีดอย่างครบวงจร โดยแบ่งการใช้งานออกเป็น 3 ส่วนหลัก ดังนี้:

1. ฝ่ายลูกค้า (Customer)

วัตถุประสงค์: สำหรับทำรายการจองคิวซักผ้า ติดตามสถานะออเดอร์ และแจ้งชำระเงิน

  • การจองคิวใช้บริการ:
    • เข้าสู่เมนู "จองคิว"
    • เลือกแพ็กเกจที่ต้องการ (เช่น ซัก-อบ 9-10 kg)
    • เลือกบริการเสริม (ถ้ามี) เช่น น้ำยาซักผ้า หรือ น้ำยาปรับผ้านุ่มสูตรต่างๆ
  • ระบบคำนวณค่าบริการรับ-ส่ง:
    • ระบบจะคำนวณค่าส่งอัตโนมัติตามระยะทางที่แอดมินได้ประเมินไว้
    • เรทราคา: ระยะทาง 0 - 1.5 กม. แรก ฟรี! ส่วนที่เกินคิดเพิ่มกิโลเมตรละ 20 บาท (เศษปัดขึ้น)
  • การชำระเงิน:
    • หากลูกค้าเลือกช่องทาง "โอนเงิน" ให้ไปที่เมนู "ออเดอร์ของฉัน" เพื่อทำการแนบสลิปหลักฐานการโอนเงินเข้าสู่ระบบ
  • การติดตามสถานะออเดอร์ (Real-time Tracking):
    • ลูกค้าสามารถตรวจสอบสถานะเสื้อผ้าของตนเองได้แบบเรียลไทม์ เช่น กำลังไปรับผ้า → รอรับผ้า → กำลังซัก/อบ → กำลังไปส่ง → เสร็จสิ้น

2. ฝ่ายผู้ดูแลระบบ (Admin)

วัตถุประสงค์: สำหรับบริหารจัดการออเดอร์ทั้งหมด ตรวจสอบการชำระเงิน และกระจายงานให้พนักงาน

  • การจัดการข้อมูลลูกค้า:
    • เมื่อมีลูกค้าใหม่สมัครสมาชิก แอดมินต้องทำการ "ประเมินระยะทาง" ในเมนูจัดการสมาชิก เพื่อให้ระบบใช้เป็นฐานข้อมูลในการคำนวณค่ารถสำหรับการจองครั้งถัดไปโดยอัตโนมัติ
  • การจ่ายงาน (Assign Job):
    • เมื่อมีออเดอร์ใหม่เข้ามา แอดมินจะตรวจสอบและเลือก "พนักงานขับรถ" จากรายชื่อที่ว่างอยู่
    • กดปุ่ม "จ่ายงาน" เพื่อส่งข้อมูลออเดอร์และพิกัดไปยังแอปพลิเคชันของไรเดอร์
  • การตรวจสอบการชำระเงิน:
    • ตรวจสอบสลิปโอนเงินที่ลูกค้าแนบมา
    • หากยอดเงินและสลิปถูกต้อง กด "อนุมัติ"
    • หากไม่ถูกต้อง กด ปฏิเสธ ระบบจะแจ้งเตือนให้ลูกค้าทำรายการส่งหลักฐานใหม่
  • การควบคุมภาพรวมระบบ (Admin Control):
    • แอดมินมีสิทธิ์แก้ไขสถานะออเดอร์แทนไรเดอร์ได้ในกรณีฉุกเฉิน
    • สามารถทำการ "ยกเลิกออเดอร์" ได้ โดยต้องระบุเหตุผลการยกเลิกทุกครั้ง

3. ฝ่ายพนักงานขับรถ/ไรเดอร์ (Driver)

วัตถุประสงค์: สำหรับการรับงาน นำทางไปยังจุดหมาย และอัปเดตสถานะการทำงานหน้างาน

  • การรับงาน:
    • ล็อกอินเข้าสู่ระบบ เมื่อแอดมินจ่ายงานมา รายการจะปรากฏในหน้า "งานของฉัน"
  • การนำทาง:
    • กดปุ่ม "นำทางไปรับ" ระบบจะเชื่อมโยงเปิด Google Maps และนำทางไปยังพิกัด GPS ที่ลูกค้าปักหมุดไว้ทันที
  • การอัปเดตสถานะการทำงาน (สำคัญ): ไรเดอร์ต้องกดเปลี่ยนสถานะตามขั้นตอนการทำงานจริง เพื่อให้ลูกค้าทราบความเคลื่อนไหว:
    1. รอรับผ้า: กดยืนยันเมื่อกำลังเดินทางไปรับ หรือรับผ้าจากลูกค้าแล้ว
    2. กำลังซัก/อบ: กดเมื่อนำผ้ามาถึงร้านและจัดส่งเข้าเครื่องซัก
    3. กำลังไปส่ง: กดเมื่อผ้าทำความสะอาดและอบแห้งเรียบร้อย และกำลังเดินทางนำไปคืนลูกค้า
    4. เสร็จสิ้น: กดเมื่อส่งมอบผ้าถึงมือลูกค้าเรียบร้อยแล้ว
  • การเก็บเงินสดหน้างาน:
    • ในกรณีที่ออเดอร์นั้นลูกค้าระบุว่าจ่าย "เงินสด" ไรเดอร์มีหน้าที่รับเงินจากลูกค้า และตรวจสอบยอดเงินให้ตรงกับค่าบริการที่สรุปไว้ในระบบ

ส่วนที่ 1: ประสบการณ์ของลูกค้า (Customer Experience)

[นาทีที่ 1:30 - 3:30]
"เริ่มต้นที่ ฝั่งลูกค้า กันก่อนครับ Washly ออกแบบมาให้ลูกค้าใช้งานง่ายที่สุด เพียงแค่กด 'จองคิว' ในระบบ ลูกค้าสามารถเลือกแพ็กเกจ เช่น ซัก-อบ 9-10 กิโลกรัม และสามารถ Customize ออเดอร์ของตัวเองได้เลย เช่น อยากได้น้ำยาซักผ้าสูตรลดกลิ่นอับ หรือน้ำยาปรับผ้านุ่มกลิ่นดอกไม้ ก็กดเลือกได้ทันที"

"ที่เจ๋งกว่านั้นคือ ระบบคำนวณค่าส่งอัตโนมัติ ครับ ระบบของเราใจดี ให้ส่งฟรีในระยะ 0-1.5 กิโลเมตรแรก ส่วนที่เกินระบบจะคำนวณแฟร์ๆ กิโลเมตรละ 20 บาท ปัดเศษขึ้นให้เรียบร้อย หมดปัญหาลูกค้าเถียงกับร้านเรื่องค่าส่งครับ"

"และในเรื่องการชำระเงิน ลูกค้าสามารถอัปโหลดสลิปผ่านเมนู 'ออเดอร์ของฉัน' ได้เลย และไฮไลท์คือ การติดตามสถานะแบบ Real-time ลูกค้าจะรู้เลยว่าตอนนี้ผ้ากำลังไปรับ กำลังซัก หรือกำลังมาส่ง ไม่ต้องนั่งเดาอีกต่อไปครับ"

[นาทีที่ 3:30 - 6:00]
"ถัดมาเป็นหัวใจของร้านครับ คือ ฝั่ง Admin แอดมินมีหน้าที่ควบคุมภาพรวมทั้งหมด เริ่มตั้งแต่ลูกค้าใหม่สมัครเข้ามา แอดมินจะทำการประเมินพิกัดระยะทาง เพื่อให้ระบบเซ็ตเป็นค่าเริ่มต้นในการคำนวณค่าส่งครั้งต่อๆ ไปครับ"

"เมื่อออเดอร์เข้ามา แอดมินจะกาง Dashboard ดูว่าตอนนี้ 'ไรเดอร์' คนไหนว่างบ้าง แล้วกดปุ่ม 'จ่ายงาน' (Assign Job) งานจะเด้งไปที่มือถือของไรเดอร์คนนั้นทันที"

"นอกจากนี้ แอดมินยังเป็นด่านตรวจความเรียบร้อย เช่น การตรวจสลิปโอนเงิน ถ้ายอดเงินเป๊ะ กด 'อนุมัติ' แต่ถ้าสลิปปลอมหรือยอดไม่ตรง กด 'ปฏิเสธ' เพื่อให้ลูกค้าส่งใหม่ได้ครับ และในกรณีฉุกเฉิน แอดมินมีสิทธิ์สูงสุดในการแก้ไขสถานะออเดอร์แทนไรเดอร์ หรือกดยกเลิกออเดอร์พร้อมระบุเหตุผลได้ ทำให้การควบคุมระบบรัดกุมมากครับ"

ส่วนที่ 3: การทำงานของพนักงานส่งผ้า (Driver/Rider Operations)

[นาทีที่ 6:00 - 8:30]
"เฟืองตัวสุดท้ายที่จะขาดไม่ได้คือ ฝั่งไรเดอร์ ครับ แอปฯ ฝั่งไรเดอร์ออกแบบมาให้ทำงานหน้างานสะดวกที่สุด เมื่อแอดมินจ่ายงานมา งานจะเด้งเข้าหน้า 'งานของฉัน' ไรเดอร์ไม่ต้องจำทางครับ แค่กดปุ่ม 'นำทางไปรับ' ระบบจะดึงพิกัด GPS ของลูกค้าไปเปิดบน Google Maps ให้นำทางทันที"

"ระหว่างการทำงาน ไรเดอร์จะทำหน้าที่อัปเดตสถานะให้ลูกค้าอุ่นใจ โดยกดเปลี่ยนสถานะตามจริง ตั้งแต่ 'รอรับผ้า' > 'กำลังซัก/อบ' (เมื่อเอาผ้าเข้าเครื่อง) > 'กำลังไปส่ง' > จนถึง 'เสร็จสิ้น'"

"และถ้ารายการไหนที่ลูกค้าเลือกจ่าย 'เงินสด' ระบบจะแจ้งเตือนไรเดอร์ชัดเจนว่า ออเดอร์นี้ต้องเก็บเงินสด ไรเดอร์ก็เช็คยอดจากในแอปฯ และเก็บเงินลูกค้าหน้าบ้านได้เลย ป้องกันการลืมเก็บเงินได้อย่าง 100% ครับ"

🔌 API Endpoints (RESTful API)

เอกสาร API สำหรับนักพัฒนาที่ต้องการเชื่อมต่อระบบ Washly กับแอปพลิเคชันภายนอก

📋 สารบัญ API

  • Authentication (การยืนยันตัวตน)
  • Customer API (ลูกค้า)
  • Admin API (ผู้ดูแลระบบ)
  • Driver API (พนักงานขับรถ)
  • Notification API (การแจ้งเตือน)

🔐 Authentication API

POST /login

คำอธิบาย: เข้าสู่ระบบสำหรับลูกค้า

Request Body:

{
  "username": "string (required)",
  "password": "string (required)"
}

Response (Success - 200):

{
  "success": true,
  "message": "เข้าสู่ระบบสำเร็จ ยินดีต้อนรับกลับมาครับ",
  "redirect": "/customer/main"
}

Response (Error - 422):

{
  "success": false,
  "errors": {
    "username": ["ชื่อผู้ใช้งาน หรือ รหัสผ่าน ไม่ถูกต้องนะ! 🥺"]
  }
}

POST /register

คำอธิบาย: สมัครสมาชิกใหม่

Request Body:

{
  "fullname": "string (required)",
  "username": "string (required, unique)",
  "phone": "string (required)",
  "address": "string (required)",
  "latitude": "number (optional)",
  "longitude": "number (optional)",
  "password": "string (required, min: 8)",
  "password_confirmation": "string (required)"
}

Response (Success - 200):

{
  "success": true,
  "message": "สมัครสมาชิกสำเร็จ! ยินดีต้อนรับครับ",
  "redirect": "/"
}

POST /logout

คำอธิบาย: ออกจากระบบ

Headers: Authorization: Bearer {token}

Response (Success - 200):

{
  "success": true,
  "message": "ออกจากระบบเรียบร้อยแล้ว",
  "redirect": "/login"
}

👤 Customer API

GET /customer/book

คำอธิบาย: แสดงหน้าฟอร์มจองคิว

Headers: Authorization: Bearer {token}

Response Data:

  • packages - รายการแพ็กเกจบริการ
  • timeSlots - ช่วงเวลาที่สามารถจองได้
  • addons - บริการเสริม (น้ำยาซัก, น้ำยาปรับผ้านุ่ม, ฯลฯ)
  • todayQueues - คิวจองวันนี้
  • tomorrowQueues - คิวจองพรุ่งนี้

POST /customer/book

คำอธิบาย: สร้างการจองคิวใหม่

Request Body:

{
  "pickup_date": "today|tomorrow (required)",
  "package_id": "integer (required)",
  "time_slot_id": "integer (required)",
  "pickup_address": "string (required)",
  "latitude": "number (optional)",
  "longitude": "number (optional)",
  "payment_method": "transfer|cash (required)",
  "customer_note": "string (optional)",
  "use_customer_detergent": "boolean (optional)",
  "use_customer_softener": "boolean (optional)",
  "addons": [
    {
      "code": "string",
      "qty": "integer (1-10)"
    }
  ],
  "extra_dry_qty": {
    "package_id": "integer"
  }
}

Response (Success - 200):

{
  "success": true,
  "message": "จองคิวสำเร็จ! หมายเลขออเดอร์ของคุณคือ ORD-20260315-3009",
  "order_number": "ORD-20260315-3009",
  "redirect": "/customer/main"
}

คำอธิบาย: ดูประวัติออเดอร์ทั้งหมดของลูกค้า

Headers: Authorization: Bearer {token}

📊 HTTP Status Codes

Code Meaning
200 OK สำเร็จ
302 Found Redirect (หลังทำรายการสำเร็จ)
401 Unauthorized ไม่ได้ล็อกอิน หรือ Token หมดอายุ
403 Forbidden ไม่มีสิทธิ์เข้าถึง (Role ไม่ตรง)
404 Not Found ไม่พบข้อมูลที่ร้องขอ
422 Unprocessable Entity Validation Error (ข้อมูลไม่ถูกต้อง)
500 Internal Server Error ข้อผิดพลาดจากเซิร์ฟเวอร์

🔑 หมายเหตุเกี่ยวกับการ Authentication

⚠️ สำคัญ: ระบบ Washly ใช้ Session-based Authentication (Laravel Session) ไม่ใช่ Token-based

หากต้องการใช้ API จากภายนอก (เช่น Mobile App) แนะนำให้ติดตั้ง Laravel Sanctum เพื่อรองรับ API Token Authentication

💡 Tip: สามารถทดสอบ API ได้ผ่าน Postman หรือ Insomnia โดยตั้งค่า Headers:

Accept: application/json
Cookie: laravel_session={session_id}
